You are a Principal Software Architect tasked with designing and building a production-ready enterprise application. Your deliverable is a complete, fully functional, deployment-ready codebase—not a prototype, demo, or scaffolding project.

**PROJECT: AI-Powered Ticket Intelligence & SLA Management Platform**

**CORE OBJECTIVE**

Build an intelligent support ticket management system that automatically ingests tickets from multiple sources (Gmail, Outlook, CSV, manual entry), analyzes each ticket using Gemini AI to determine complexity and business impact, predicts SLA breach risk, automatically assigns tickets to optimal engineers based on skill match and workload, tracks SLA compliance in real-time, and provides role-specific dashboards with actionable analytics.

The primary success metric: **Reduce SLA breaches through intelligent ticket analysis, prioritization, and assignment.**

---

- **Frontend:** Streamlit (enterprise SaaS dashboard style—minimal, professional, no unnecessary animations)
- **Backend:** FastAPI with production-grade error handling and logging
- **Database:** PostgreSQL via SQLAlchemy ORM
- **Validation:** Pydantic for all inputs
- **AI:** Gemini API for ticket analysis
- **Authentication:** JWT-based role access control
- **Notifications:** Discord Webhooks

- **Deployment:** Render + Neon PostgreSQL

- Receive critical alerts

**TEAM MEMBER**

**AI ANALYSIS ENGINE**

For every ingested ticket, call Gemini API with ticket subject and body. Gemini must return a JSON response with exactly these fields:

```
{
  "category": "[one of: Database, Network, Security, Infrastructure, Cloud, Application, Performance, Authentication, Other]",
  "complexity": "[1-5 scale: 1=Very Simple, 2=Simple, 3=Moderate, 4=Complex, 5=Critical]",
  "priority": "[P1/P2/P3/P4]",
  "business_impact": "[Low/Medium/High]",
  "recommendation": "[actionable text]"
}
```

Store this analysis in the database immediately after creation.

---

**SLA ENGINE**

Calculate SLA deadlines and breach status for every ticket using these thresholds:
- P1: 4 hours
- P2: 8 hours
- P3: 24 hours
- P4: 48 hours

For each ticket, compute: SLA Deadline timestamp, Remaining Time (countdown), and Breached status (boolean).

---

**RISK PREDICTION ENGINE**

Generate a Risk Score (0-100) for every ticket based on:
- Remaining time until SLA breach
- Priority level
- Complexity rating
- Business impact level
- Current ticket volume

Risk Level Classifications:
- 0-30: Low
- 31-60: Medium
- 61-80: High
- 81-100: Critical

Store risk scores and update in real-time as tickets progress.

---

**SMART ASSIGNMENT ENGINE**

Automatically assign each ticket to the engineer with the highest probability of resolution before SLA breach. Assignment factors (in priority order):
1. Skill match (engineer's historical category expertise)
2. Current active ticket count (lower is better)
3. Current workload (total complexity points assigned)
4. Priority level (P1 critical tickets go to top performers)
5. Open ticket count

**Assignment must be automatic with no manual approval step.** Implement this as a background job that runs when tickets are created and when status changes occur.

---

**EMAIL INGESTION WORKFLOW**

Implement automatic polling (not a manual sync button) every 5 minutes for Gmail and Outlook:

Email → Extract Subject + Body → Create Ticket Record → Trigger AI Analysis → Calculate SLA → Generate Risk Score → Auto-Assign → Store in Database → Trigger Notifications

Handle email authentication via OAuth2. Store email credentials securely using environment variables. Process failures gracefully with retry logic and detailed logging.

---

**TICKET LIFECYCLE**

Tickets follow this state machine:
- Open → In Progress → Resolved → Closed

Only Team Leads can transition Resolved → Closed. Team Members can transition Open → In Progress and In Progress → Resolved.

**AI INSIGHTS ENGINE**

When a Team Member resolves a ticket, automatically generate and display personal performance insights on their dashboard:

```
Resolved Tickets: [count]

Network Tickets: [count]
Security Tickets: [count]
[...all categories...]
Most Frequent Category: [name]
Average Resolution Time: [hours]

```

Update these insights in real-time as new resolutions occur.

**DISCORD NOTIFICATIONS**

Send alerts to both the assigned engineer AND the Team Lead whenever:
1. A critical ticket is created (P1 + High Risk)
2. Risk Score exceeds 80
3. Remaining SLA time drops below 30 minutes
4. SLA is breached

Alert format must include: Ticket ID, Category, Priority, Risk Score, Assigned Engineer, Gemini Recommendation.
